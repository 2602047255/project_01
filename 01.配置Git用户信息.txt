安装完 Git 之后，要做的第一件事就是设置自己的用户名和邮箱。
因为通过 Git 对项目进行版本管理的时候，Git 需要使用这些基本信息，来记录是谁对项目进行了操作。
配置基本信息：
    git config --global user.name "用户名"
    git config --global user.email "邮箱"

配置后的基本信息，会被写入到 C:/Users/用户名文件夹/.gitconfig 文件里面，这个文件叫做：“Git 的全局配置文件”。
查看所有的全局配置项：
    git config --list --global

查看指定的全局配置项：
    git config user.name
    git config user.email

获取帮助信息，在无需联网的情况下，可以在浏览器中打开帮助手册
获取帮助信息：
    git help 指定命令
    或者
    git 指定命令 -h ----- 在命令窗口展示

如果有一个尚未进行版本控制的项目目录，想要用 Git 来控制它，需要在该目录中初始化仓库
初始化仓库：
    git init

查看文件处于什么状态：
    git status
    或者
    git status -s ----- 以精简的方式展示

当文件被跟踪之后，会处于暂存状态
跟踪新文件：
    git add 文件名
